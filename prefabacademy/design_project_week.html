Week 3 : Project - Fab-GamePad

This week we have to design and implement a project using atleast 3 machines.

I lagged behind with the project, but was able to catch up during the following days. The entire project documentation have been compiled in this page.

3D Designing the GamePad in Antimony

Antimony is a very powerful software for Parametric Hierarchical CAD modelling. I started of imagining the Fab-GamePad as exactly the same found in the commercial game stations. Soon, I found out it would be a challenge to design it in Antimony. See below for the screenshot for one such try.

I soon came to realise my mistake in modelling Fab-GamePad exactly like the gamepads available. I then decided to take full advantage of the power of Antimony by relying on as many symmetric shapes as possible. Here is the final design for the top portion of the gamepad.

I exported the file as .stl and made the .gcode file from Cura in the Normal Print mode. Time for some 3D printing!

Found that there was an imperfection in the print in the areas where the button goes. This was due to a small provision I gave whuch did not have any support. Need to improve the design in the next iteration.

Designing the GamePad circuit in Kokopelli

I liked the entire concept of designing a circuit using code. I decided to go for a simple circuit inspired from Neil's HelloFTDI board. First I removed the crystal, so as to get more free input pins. The GamePad has 7 buttons, 4 for direction controls and the rest for miscellaneous controls. Here are the original size design files :

Programming the Electronics

Since I did not use any crystal, I should burn the bootloader from Arduino with the proper setting as shown in figure otherwise the board may not respond.

When tested one of the buttons with the standard code for input push-button, I found randomly fluctuating values. Unfortunately, instead of wiring the button between Ground and a Pin, I did it between VCC and the Pin. This rendered it impossible to use the Pull-Up resistor (since such a circuit calls for pull-down instead) and I had to do some patch work to add external 1k resitors as pull-down resistors

Here's the final code for the board. 


int buttonA = 4;
int buttonB = 3;
int buttonC = 2;
int buttonU = 9;
int buttonD = 8;
int buttonL = 7;
int buttonR = 10;

int a,b,c,u,d,l,r;

#include 

SoftwareSerial mySerial(0, 1); // RX, TX

void setup() {
  
  pinMode(buttonA,INPUT);
  pinMode(buttonB,INPUT);
  pinMode(buttonC,INPUT);
  pinMode(buttonU,INPUT);
  pinMode(buttonD,INPUT);
  pinMode(buttonL,INPUT);
  pinMode(buttonR,INPUT);
  // set the data rate for the SoftwareSerial port
  mySerial.begin(9600);
}

void loop() { 

  a = digitalRead(buttonA);
  b = digitalRead(buttonB);
  c = digitalRead(buttonC);
  u = digitalRead(buttonU);
  d = digitalRead(buttonD);
  l = digitalRead(buttonL);
  r = digitalRead(buttonR);

  if(a==1)
    mySerial.println('a');
  else if(b==1)
    mySerial.println('b');
  else if(c==1)
    mySerial.println('c');
  else if(u==1)
    mySerial.println('u');
  else if(d==1)
    mySerial.println('d');
  else if(l==1)
    mySerial.println('l');
  else if(r==1)
    mySerial.println('r');

}

Finally, I also printed out a label for the Fab-GamePad on the Vinyl Cutter.
