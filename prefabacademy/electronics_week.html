Week 4 : The world of Electronics

Week 4 involves getting to know about Electronics design, production and programming.

Week 4 was all about electronics design and programming. We were guided through the software setup for kokopelli, the software that allows to design circuits using python code! The weeks activities included installing and getting familiar with :

  • Installation and working of Kokopelli software
  • Adding LED and Button to Neil's HelloFTDI board
  • Milling Hello FTDI using Modella
  • Programming Hello FTDI

Day 1

Day 1 started off with Francisco going through the FabLab presentation, where he explained the need for digital manufacturing and how our choices are impacting the dumping of tonnes of waste due to mass production.

We proceeded with installation of Kokopelli, the software which uses code to produce electronic circuit design. It revolves around the concept of adding programmability into every layer of digital fabrication, and research is going on to even make matter programmable.

The installation proved to be a task in itself. I used the VirtualBox Ubuntu setup that was done in the first week. I downloaded the koko_retro.zip file which contained the source code for the older version of kokopelli since Neil’s part library works only with the old version. To compile kokopelli, the following steps were followed : 

cd koko_retro
make clean
make fab

After successful compiling, to open kokopelli, i run : 

./kokopelli -r

However, the program didn't run as expected and showed a blank view even though a sample program for a circle was pre-loaded. I tried make clean, make fab again to no avail. I finally tracked down the problem to be a missing library and it worked fine after I did some pip install taking cue from the github page of kokopelli.

Then Francisco showed how Neil had made the entire inventory library in code. He even programmed the Alphabets instead of using a font!

We opened Neil's hello.ftdi circuit and added an led and button to it, following this excellent tutorial on kokopelli by Francisco. The whole exercise opened my eyes to the impact programmability has. Just changing a few parameters, and adding some components means easy and fast customisation of circuits. Day 2

Day 2 was for actually milling out the circuit that we had built yesterday using Kokopelli. Since there is only one Roland Modella available, we took turns in milling our boards. While I was waiting for my turn, I thought of tweaking and adding some more components to it. First I looked into the available sensors in the lab. I found a light sensor, which was a PHOTODARLINGTON NPN CLR PLCC-2. I thought it would be great if I could add this as an ambient light sensor to switch on the LED when it was dark and switch off the LED when it was bright. This would allow me to learn more about analog sensors along with the Push-Button, which is a digital Sensor. I looked into the datasheet of the Phototransistor and found that adding it to the existing circuit should be very easy.

I drew a rough circuit diagram on paper to visualise the connections needed. Then one by one I placed the components and joined them to the existing circuit defining wires between VCC, Sensor terminals, Ground and one of the IC analog input.

Day 5

On Day 5, Luciano came along with our second instructor Francisco, who would be engaging us in the last week of the Pre-FabAcademy. Luciano continued with the previous day's topic of CAD Design with Rhino. An assignment was given the previous day and the submission was discussed. I tried out the assignment in the class, trying to catch up yesterday's work.

Francisco introduced yet another CAD software called Antimony. He explained the key difference between Antimony and other CAD software. Antimony is the only software that gives exact geometric representation of a 3D model instead of finite element approximations. The distinguishing features of Antimony are :

  • Geometric Function representation
  • Hierarchical Design
  • Parametric Design

I liked Antimony for its simplicity and focus on mathematical accuracy. The software though not suited for aethetic/organic design, is very powerful for creating mechanical component design. Francisco showed us a demo for a bit-holder in Antimony. I went ahead and made a cylindrical version of the bit-holder.

I caught up with yesterday's Rhino with some help from Sibu, one of the Pre-FabAcademy participants. I plan to look more into Rhino during the weekend.
